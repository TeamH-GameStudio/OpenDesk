# OpenDesk 유저 스토리

## 페르소나

| 역할 | 설명 |
|------|------|
| **비개발자 PM** | AI에 업무를 시키고 싶지만 터미널이 낯선 직장인 |
| **1인 창업자** | Claude Code로 개발을 위임하고 결과만 받고 싶은 사람 |
| **개발자** | 여러 AI 에이전트를 동시에 돌리며 상태를 모니터링하는 사람 |

### Epic 1: Cold Start — 첫 실행 & 초기 세팅

---

**Title:** 미들웨어 게이트웨이 연결 설정

**Description:** 비개발자 PM으로서, OpenDesk와 Claude Code 에이전트를 중계하는 게이트웨이 서버 주소를 앱에 입력하고 싶다. 그래야 앱이 에이전트의 상태를 실시간으로 받아오고, 반대로 지시를 보낼 수 있기 때문이다.

**Design:** [Figma — Onboarding Step 1: Gateway Connection Setup]

**Acceptance Criteria:**
1. 최초 실행 시 게이트웨이 연결 설정 화면이 자동으로 표시된다.
2. 서버 주소(host) 및 포트(port) 입력 필드가 제공된다 (기본값: `localhost:3000`).
3. "연결 테스트" 버튼 클릭 시 WebSocket 핸드셰이크를 시도하고 결과를 즉시 표시한다.
4. 연결 성공 시 초록색 불과 "연결됨" 텍스트가 표시된다.
5. 연결 실패 시 에러 메시지(예: "서버에 연결할 수 없어요. 주소를 확인해주세요")와 재시도 버튼이 표시된다.
6. 연결 정보는 앱 재시작 후에도 유지되며, 설정 화면에서 언제든지 변경 가능하다.
---

**Title:** 에이전트 기본 정보 설정

**Description:** 1인 창업자로서, 내 AI 에이전트에게 이름과 역할을 부여하고 싶다. 오피스에 있는 치비 캐릭터가 누구인지 알 수 있기 때문이다.

**Design:** [Figma — Onboarding Step 2: Agent Profile]

**Acceptance Criteria:**
1. 에이전트 이름 입력 필드가 제공된다 (기본값: "My Assistant").
2. 역할 선택지가 제공된다: 개발, 기획, 리서치, 디자인, 기타.
3. 역할에 따라 치비 캐릭터의 의상/색상이 미리보기로 변경된다.
4. 설정 완료 시 "오피스로 들어가기" 버튼이 활성화된다.
5. 설정은 나중에 언제든지 에이전트 카드에서 수정 가능하다.

### Epic 2: 세션 선택 — 에이전트 감지 & 입장

---

**Title:** 세션 자동 감지 및 리스트 표시

**Description:** 개발자로서, 게이트웨이에 접속된 활성 Claude Code 세션 목록을 실시간으로 받아오고 싶다. 어떤 작업이 돌아가고 있는지 한눈에 보고 싶기 때문이다.

**Design:** [Figma — Session List Screen]

**구현 방향:** 파일 시스템 폴링 대신 WebSocket 이벤트 구독 방식. 게이트웨이 연결 직후 `session_list` 이벤트로 초기 목록을 수신하고, 이후 `session_created` / `session_closed` 이벤트로 실시간 갱신.

**Acceptance Criteria:**
1. 게이트웨이 연결 즉시 `session_list` 이벤트를 수신하여 현재 활성 세션을 리스트에 표시한다.
2. 각 세션 카드에 세션 ID, 시작 시간, 현재 상태(Working/Idle/Error), 네트워크 핑(ms)이 표시된다.
3. `session_created` 이벤트 수신 시 새 세션 카드가 리스트 최상단에 즉시 추가된다 (폴링 없음).
4. `session_closed` 이벤트 수신 시 해당 카드가 리스트에서 제거된다.
5. 세션이 없는 경우 "아직 실행 중인 에이전트가 없어요" 빈 상태 화면이 표시된다.
6. 세션 카드 클릭 시 해당 에이전트가 있는 오피스 씬으로 전환된다.
7. 여러 세션이 있을 때 가장 최근 활동 순으로 정렬된다.

---

**Title:** 에이전트 상태 뱃지 표시

**Description:** 비개발자 PM으로서, 각 에이전트가 지금 뭘 하고 있는지 세션 목록에서 바로 알고 싶다. 어느 에이전트에게 말을 걸어야 할지 판단하기 위해서이다.

**상태 기술 정의 (게이트웨이 스트림 기반):**
- **Working**: 게이트웨이로부터 `type: "tool_call"` JSON 이벤트가 수신된 순간.
- **Thinking**: `type: "thinking"` 이벤트 수신 중 — 모델이 텍스트를 생성 중이나 아직 툴을 호출하지 않은 구간.
- **Idle**: 마지막 이벤트로부터 30초 이상 경과, 또는 `type: "task_completed"` 수신.
- **Error**: `type: "error"` 이벤트 수신.
- **Sleeping**: 마지막 이벤트로부터 5분 이상 경과.

**Acceptance Criteria:**
1. 세션 카드에 현재 상태 뱃지가 표시된다: 🟢 Working / 🟡 Thinking / ⚪ Idle / 🔴 Error / 💤 Sleeping.
2. 상태 뱃지는 게이트웨이 WebSocket 스트림의 `type` 필드 분류 결과에 따라 실시간 업데이트된다.
3. 마지막 활동 시간이 "3분 전", "방금" 형식으로 표시된다.
4. Error 상태일 때 뱃지가 깜빡이는 시각적 효과가 있다.

---

### Epic 3: 메인 씬 — 오피스 & 캐릭터 시각화

---

**Title:** 에이전트 상태별 캐릭터 애니메이션

**Description:** 비개발자 PM으로서, AI 에이전트가 지금 무슨 일을 하는지 캐릭터 움직임만 봐도 알고 싶다. 로그를 읽지 않아도 직관적으로 상태를 파악하기 위해서이다.

**Design:** [Figma — Main Office Scene]

**Acceptance Criteria:**
1. 게이트웨이 스트림 이벤트의 `type` 필드 분류 결과에 따라 6가지 애니메이션이 전환된다: Idle / Working / Thinking / Happy / Error / Sleeping.
2. 상태 전환은 `Animator.CrossFade()`를 사용하여 부드럽게 블렌딩된다 (전환 시간 0.2~0.3초, 끊김 없음).
3. Working 상태: 키보드 타이핑 애니메이션 + 모니터 화면 글씨 깜빡임.
4. Thinking 상태: 머리 위 말풍선 + "..." 애니메이션.
5. Happy 상태: 작업 완료 시 짧은 점프 + 반짝임 이펙트.
6. Error 상태: 캐릭터 머리 위 빨간 느낌표 + 진동 효과.
7. Sleeping 상태: zzz 말풍선, 비활성 시 자동 전환 (5분 무활동 기준).

---

**Title:** 실시간 작업 툴팁 표시

**Description:** 1인 창업자로서, 캐릭터 위에 지금 실행 중인 작업이 뭔지 간략히 보고 싶다. 에이전트가 뭘 하는지 화면을 전환하지 않고 바로 알기 위해서이다.

**Acceptance Criteria:**
1. Working 상태일 때 캐릭터 위에 현재 툴콜 이름이 말풍선으로 표시된다 (예: "파일 읽는 중...").
2. 말풍선 텍스트는 최대 20자로 잘리고 "..." 처리된다.
3. 말풍선은 마우스 호버 시에도 표시된다.
4. 작업이 완료되면 말풍선이 페이드 아웃된다.

### Epic 4: 작업 내역 명세

---

**Title:** 태스크 레벨 로그 조회

**Description:** 개발자로서, 에이전트가 어떤 툴을 어떤 순서로 실행했는지 상세히 보고 싶다. 작업이 예상대로 진행되었는지 검증하기 위해서이다.

**Design:** [Figma — Task Log Panel]

**구현 방향:** 게이트웨이 WebSocket 스트림에서 수신된 이벤트를 Unity 메모리(List<AgentEvent>)에 누적 적재. 세션 종료 후 장기 보관이 필요한 경우 미들웨어 DB에서 `GET /sessions/{id}/logs` API로 조회.

**Acceptance Criteria:**
1. 오피스 씬에서 캐릭터 클릭 시 작업 내역 패널이 슬라이드인으로 열린다.
2. 툴콜 목록이 시간순으로 표시된다: 툴 이름, 상태(성공/실패/진행중), 타임스탬프.
3. 각 항목 클릭 시 입력 파라미터와 출력 요약이 펼쳐진다.
4. 성공은 초록, 실패는 빨강, 진행 중은 파란색으로 구분된다.
5. 새 툴콜은 패널이 열려 있을 때 상단에 실시간으로 추가된다 (WebSocket 스트림 실시간 반영).
6. "처음부터 보기" / "최신부터 보기" 토글이 제공된다.

---

**Title:** 작업 요약 카드 표시

**Description:** 비개발자 PM으로서, 복잡한 로그 없이 에이전트가 오늘 뭘 했는지 한 줄 요약으로 보고 싶다. 기술 내용 없이 성과만 파악하기 위해서이다.

**Acceptance Criteria:**
1. 작업 내역 패널 상단에 "오늘의 작업 요약" 카드가 표시된다.
2. 완료된 툴콜 수, 총 소요 시간, 마지막 완료 작업명이 표시된다.
3. 요약 텍스트는 비기술적 언어로 작성된다 (예: "파일 12개 수정, 코드 검토 완료").
4. 에러 발생 건수가 있으면 경고 아이콘과 함께 표시된다.

### Epic 5: 에이전트와 1대1 대화

---

**Title:** 채팅으로 작업 지시

**Description:** 비개발자 PM으로서, 텔레그램처럼 메시지를 보내서 에이전트에게 작업을 지시하고 싶다. 터미널이나 CLI를 쓰지 않아도 AI에게 일을 시키기 위해서이다.

**Design:** [Figma — Chat Interface]

**구현 방향:** 게이트웨이의 `send_message` 엔드포인트로 메시지를 전송하면, 게이트웨이가 해당 Claude Code 세션으로 포워딩한다. 응답은 기존 WebSocket 스트림을 통해 청크(chunk) 단위로 수신되며, 별도의 프로세스 생성이나 stdin 직접 제어가 불필요하다.

```
Unity 채팅창 → WebSocket send_message → Gateway → Claude Code 세션
Unity 채팅창 ←  WebSocket stream chunk ← Gateway ← Claude Code 응답
```

**Acceptance Criteria:**
1. 캐릭터 더블클릭 또는 채팅 버튼 클릭 시 채팅창이 열린다.
2. 메시지 입력 후 Enter 또는 전송 버튼으로 게이트웨이 `send_message`에 전송된다.
3. 전송된 메시지는 오른쪽 말풍선으로 표시된다.
4. 에이전트의 응답은 WebSocket 스트림 청크를 수신하여 왼쪽 말풍선에 실시간으로 타이핑 효과로 표시된다.
5. 지시 전송 즉시 캐릭터가 Thinking 애니메이션으로 전환된다.
6. 채팅창은 오피스 씬 위에 오버레이로 열리며, 배경의 캐릭터 애니메이션이 계속 보인다.
7. 세션 연속성이 보장된다 — 이전 대화 컨텍스트가 게이트웨이 세션에 유지된다.

---

**Title:** 작업 완료 알림 수신

**Description:** 1인 창업자로서, 에이전트가 지시한 작업을 완료하면 알림을 받고 싶다. 앱을 계속 보고 있지 않아도 작업 완료를 알기 위해서이다.

**Acceptance Criteria:**
1. 작업 완료 시 OS 네이티브 알림(토스트)이 표시된다.
2. 알림에 에이전트 이름과 완료 작업 요약이 포함된다.
3. 알림 클릭 시 앱이 포커스되고 해당 에이전트의 채팅창이 열린다.
4. 채팅창에 완료 메시지가 Happy 이모지와 함께 표시된다.
5. 앱이 최소화된 상태에서도 알림이 전달된다.
6. 알림 허용/차단은 설정에서 변경 가능하다.

---

**Title:** 작업 현황 중간 보고 수신

**Description:** 비개발자 PM으로서, 긴 작업이 진행되는 동안 중간중간 뭘 하고 있는지 메시지로 받고 싶다. 에이전트가 멈췄는지 아니면 열심히 하고 있는지 알기 위해서이다.

**Acceptance Criteria:**
1. 툴콜 단위로 진행 상황이 채팅 메시지로 표시된다 (예: "파일 분석 중...").
2. 메시지는 실시간 스트리밍으로 표시된다.
3. 각 진행 메시지에 경과 시간이 표시된다 (예: "시작 후 2분").
4. 에러 발생 시 빨간색 에러 메시지가 채팅에 표시되고 에이전트 표정이 Error로 변환된다.
5. 장시간 무응답(3분)이면 "⏳ 열심히 생각 중이에요..." 자동 메시지가 표시된다.

### Epic 6: 멀티 에이전트 오피스

---

**Title:** 여러 에이전트 동시 시각화

**Description:** 개발자로서, 여러 Claude Code 세션을 동시에 오피스에 띄워놓고 싶다. 병렬로 돌리는 에이전트들의 상태를 한눈에 보기 위해서이다.

**Acceptance Criteria:**
1. 최대 4개의 에이전트가 오피스 씬에 동시에 표시된다.
2. 각 캐릭터는 독립적인 책상/구역을 차지한다.
3. 각 캐릭터 위에 에이전트 이름 레이블이 표시된다.
4. 모든 캐릭터의 상태 애니메이션이 독립적으로 동작한다.
5. 5번째 에이전트부터는 오피스 확장 또는 추가 구매 안내가 표시된다.

---

**Title:** 에이전트 간 협업 시각화

**Description:** 개발자로서, 에이전트들이 같은 파일을 다루거나 연관 작업을 할 때 시각적으로 연결되어 보이길 원한다. 에이전트 간 관계를 직관적으로 파악하기 위해서이다.

**Acceptance Criteria:**
1. 같은 파일/폴더를 동시에 작업 중인 에이전트 사이에 점선 연결이 표시된다.
2. 한 에이전트가 다른 에이전트의 출력을 입력으로 받으면 화살표 효과가 표시된다.
3. 연결선 위에 공유 리소스 이름이 말풍선으로 표시된다.

**확장 방향 (Post-Post-MVP):** 에이전트 간 연결 시각화 구조는 추후 Agentic Workflow — 에이전트끼리 순차/병렬로 대화하는 워크플로우 — 를 시각화하는 기반으로 확장 가능하다.

### Epic 7: 바이럴 트리거

---

**Title:** 작업 완료 스크린샷 공유

**Description:** 1인 창업자로서, AI 에이전트가 작업을 완료했을 때 오피스 씬 스크린샷을 SNS에 공유하고 싶다. 내 AI 비서 자랑을 하고 싶기 때문이다.

**Acceptance Criteria:**
1. Happy 애니메이션 피크 타이밍에 "공유하기" 버튼이 잠깐 표시된다.
2. 버튼 클릭 시 오피스 씬 + 완료 작업 요약이 합성된 이미지가 생성된다.
3. 이미지에 "Made with OpenDesk" 워터마크가 포함된다.
4. 클립보드 복사, 파일 저장, Twitter/X 공유 옵션이 제공된다.
5. 공유 이미지는 1200×630px (OG 이미지 사이즈)로 생성된다.

**Title:** 오피스 꾸미기 (가구/아이템)

2. 기본 아이템(화분, 메모판, 커피컵 등) 10종이 무료로 제공된다.

4. 배치 레이아웃은 저장되어 재시작 후에도 유지된다.
5. 프리미엄 아이템 팩은 인앱 결제로 제공된다.

**Title:** 에이전트 레벨업 시스템

**Description:** 비개발자 PM으로서, 에이전트가 작업을 많이 할수록 캐릭터가 성장하는 걸 보고 싶다. 앱을 계속 쓰고 싶은 이유가 생기기 때문이다.

**Acceptance Criteria:**
1. 누적 툴콜 수/작업 시간에 따라 에이전트 레벨이 올라간다.
2. 레벨업 시 특별 애니메이션과 알림이 표시된다.
3. 레벨에 따라 캐릭터 의상/액세서리가 언락된다.
4. 레벨 및 획득 아이템은 로컬에 저장된다.
5. 레벨은 앱 재설치 시 초기화되지 않도록 클라우드 백업 옵션이 제공된다 (Post-Post-MVP).

### Epic 8: 고급 모니터링

---

**Title:** 작업 통계 대시보드

**Description:** 개발자로서, 에이전트가 이번 주에 얼마나 일했는지 통계로 보고 싶다. 에이전트 활용 패턴을 파악하고 비용을 최적화하기 위해서이다.

**Acceptance Criteria:**
1. 주간/월간 툴콜 수, 작업 시간, 완료 태스크 수가 차트로 표시된다.
2. 가장 많이 사용된 툴 TOP 5가 표시된다.
3. 에러 발생 패턴 (시간대, 툴 종류별)이 표시된다.
4. 데이터는 로컬 SQLite에 저장된다.
5. CSV 내보내기가 지원된다.

---

**Title:** 에러 알림 및 자동 재시도

**Description:** 개발자로서, 에이전트가 에러로 멈췄을 때 자동으로 재시도하거나 즉시 알림을 받고 싶다. 에이전트가 멈춘 걸 몇 시간 뒤에 발견하는 상황을 막기 위해서이다.

**Acceptance Criteria:**
1. Error 상태 감지 시 5초 내 OS 알림이 전송된다.
2. 알림에 에러 요약과 "재시도" 액션 버튼이 포함된다.
3. 에러 유형에 따라 자동 재시도 횟수를 설정할 수 있다 (0~3회).
4. 재시도 간격은 설정 가능하다 (10초 / 30초 / 1분).
5. 모든 재시도 실패 시 에이전트 상태가 Error로 고정되고 추가 알림이 전송된다.

## MVP 범위 요약

```
🟢 MVP (v1.0)                           [통신 방식: WebSocket ↔ Gateway]
├── Cold Start
│   ├── 게이트웨이 연결 설정 (host:port 입력 → WebSocket 핸드셰이크)
│   └── 에이전트 기본 정보 설정
├── 세션 선택
│   ├── 세션 자동 감지 (session_list / session_created 이벤트 구독)
│   └── 에이전트 상태 뱃지 (type 필드 실시간 분류)
├── 메인 오피스 씬
│   ├── 상태별 캐릭터 애니메이션 (6종, 스트림 이벤트 트리거)
│   └── 실시간 작업 툴팁
├── 작업 내역 명세
│   ├── 태스크 레벨 로그 조회 (메모리 누적 / DB API 조회)
│   └── 작업 요약 카드
└── 1대1 대화
    ├── 채팅으로 작업 지시 (send_message → Gateway 포워딩)
    ├── 작업 완료 알림
    └── 작업 현황 중간 보고 (스트림 청크 실시간 표시)

🔵 Post-MVP (v1.x ~ v2.0)
├── 멀티 에이전트 오피스 (4명 동시)
├── 공유 스크린샷 (바이럴)
├── 오피스 꾸미기
├── 레벨업 시스템
├── 통계 대시보드
└── 에러 자동 재시도
```
